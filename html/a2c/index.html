演员-评论员算法 (Actor-Critic) - A2C

本文件是蘑菇书EasyRL第九章的PyTorch实现。

Actor-Critic算法结合了策略梯度(Actor)和价值函数(Critic)的优点： - Actor (演员)：负责学习策略 πθ(a|s)，输出动作 - Critic (评论员)：负责学习价值函数 Vϕ(s)，评估Actor的表现

优势函数

优势函数衡量在状态 s 下采取动作 a 相对于平均水平的优势：

Aπ(s,a)=Qπ(s,a)−Vπ(s)

其中： - Qπ(s,a) 是动作价值函数 - Vπ(s) 是状态价值函数

优势函数可以通过TD残差估计：

A(st,at)≈δt=rt+γV(st+1)−V(st)

Actor-Critic目标函数

Actor的更新方向由优势函数加权：

∇θJ≈𝔼t[A(st,at)∇θlogπθ(at|st)]

Critic通过最小化TD误差来学习价值函数：

Lcritic=𝔼t[(rt+γV(st+1)−V(st))2]

A2C (Advantage Actor-Critic)

A2C是同步的Actor-Critic算法，多个并行环境同时收集经验， 然后同步更新所有环境的策略和价值函数。

A2C算法流程

  1. 每个并行环境运行 n 步，收集 (st,at,rt,st+1)
  2. 计算优势函数（使用GAE或TD残差）
  3. 计算Actor损失和Critic损失
  4. 同步更新所有环境的网络参数

损失函数

总损失由三部分组成：

L=Lactor+c1Lcritic−c2ℋ

其中： - Lactor=−𝔼t[A(st,at)logπθ(at|st)] — Actor损失 - Lcritic=𝔼t[(Gt−V(st))2] — Critic损失 - ℋ=𝔼t[ℋ(πθ(·|st))] — 策略熵（鼓励探索） - c1,c2 是系数

蘑菇书中的Actor-Critic

根据蘑菇书第九章，Actor-Critic的核心思想是： - Actor根据策略 πθ(a|s) 选择动作 - Critic评估当前状态的价值 V(s) - 使用Critic的输出作为基线，降低策略梯度的方差

策略梯度更新：

θ←θ+α∇θlogπθ(at|st)A(st,at)

价值函数更新：

ϕ←ϕ−β∇ϕ(Gt−Vϕ(st))2

Actor-Critic共享网络

共享特征提取器的Actor-Critic网络： - 共享层：提取状态特征 - Actor头：输出策略分布 πθ(a|s) - Critic头：输出状态价值 Vϕ(s)

前向传播。

参数： - state: s — 状态

返回值： - action_probs: πθ(a|s) — 动作概率分布 - state_value: Vϕ(s) — 状态价值

评估给定状态-动作对。

用于计算 logπθ(at|st) 和策略熵。

参数： - state: s — 状态 - action: a — 动作

返回值： - log_prob: logπθ(a|s) — 动作的对数概率 - state_value: Vϕ(s) — 状态价值 - entropy: ℋ(πθ(·|s)) — 策略熵

1class ActorCriticNetwork(nn.Module):
2
3    def __init__(self, state_dim: int, action_dim: int, hidden_dim: int = 256):
4        super().__init__()
5        # 共享特征提取器
6        self.feature = nn.Sequential(
7            nn.Linear(state_dim, hidden_dim),
8            nn.ReLU(),
9            nn.Linear(hidden_dim, hidden_dim),
10            nn.ReLU(),
11        )
12
13        # Actor头：输出动作分布 \pi_\theta(\cdot|s)
14        self.actor = nn.Linear(hidden_dim, action_dim)
15
16        # Critic头：输出状态价值 V_\phi(s)
17        self.critic = nn.Linear(hidden_dim, 1)
18
19    def forward(self, state: torch.Tensor):
20        features = self.feature(state)
21
22        # Actor: \pi_\theta(a|s) = \text{softmax}(f_\theta(s))
23        action_logits = self.actor(features)
24        action_probs = torch.softmax(action_logits, dim=-1)
25
26        # Critic: V_\phi(s)
27        state_value = self.critic(features).squeeze(-1)
28
29        return action_probs, state_value
30
31    def evaluate(self, state: torch.Tensor, action: torch.Tensor):
32        features = self.feature(state)
33        action_logits = self.actor(features)
34        state_value = self.critic(features).squeeze(-1)
35
36        # 计算动作概率和对数概率
37        action_probs = torch.softmax(action_logits, dim=-1)
38        action_log_probs = torch.log(action_probs + 1e-8)
39
40        # 选择给定动作的对数概率
41        log_prob = action_log_probs.gather(-1, action.unsqueeze(-1)).squeeze(-1)
42
43        # 计算策略熵 \mathcal{H}(\pi) = -\sum_a \pi(a) \log \pi(a)
44        entropy = -(action_probs * action_log_probs).sum(dim=-1)
45
46        return log_prob, state_value, entropy

A2C损失函数

A2C的总损失由三部分组成：

L=Lactor+c1Lcritic−c2ℋ

其中： - Lactor=−𝔼[A(st,at)logπθ(at|st)] - Lcritic=𝔼[(Gt−V(st))2] - ℋ=𝔼[ℋ(πθ(·|st))]

优势函数通过TD残差或GAE估计：

A(st,at)=Gt−V(st)

计算A2C损失。

参数映射： - log_probs: logπθ(at|st) — 动作的对数概率 - state_values: Vϕ(st) — 状态价值 - returns: Gt — 折扣累积回报 - entropy: ℋ(πθ(·|st)) — 策略熵

返回值： - Actor损失 - Critic损失 - 熵损失 - 总损失

1class A2CLoss(nn.Module):
2
3    def __init__(self, value_loss_coef: float = 0.5, entropy_coef: float = 0.01):
4        super().__init__()
5        self.value_loss_coef = value_loss_coef
6        self.entropy_coef = entropy_coef
7
8    def forward(
9        self,
10        log_probs: torch.Tensor,
11        state_values: torch.Tensor,
12        returns: torch.Tensor,
13        entropy: torch.Tensor,
14    ) -&gt; tuple:
15        # 优势函数：A(s_t, a_t) = G_t - V(s_t)
16        # detach防止梯度通过价值函数流向Actor
17        advantages = returns - state_values.detach()
18
19        # Actor损失：-\mathbb{E}[A(s_t, a_t) \log \pi_\theta(a_t|s_t)]
20        # 负号因为优化器执行最小化
21        actor_loss = -torch.mean(log_probs * advantages)
22
23        # Critic损失：\mathbb{E}[(G_t - V(s_t))^2]
24        critic_loss = torch.mean((returns - state_values) ** 2)
25
26        # 熵损失：鼓励探索
27        entropy_loss = -torch.mean(entropy)
28
29        # 总损失：L = L_{actor} + c_1 L_{critic} - c_2 \mathcal{H}
30        total_loss = actor_loss + self.value_loss_coef * critic_loss + self.entropy_coef * entropy_loss
31
32        return actor_loss, critic_loss, entropy_loss, total_loss

计算TD目标值

Gt=rt+γV(st+1)×(1−done)

如果episode结束（done=True），则 Gt=rt。

参数： - rewards: rt — 每步奖励 - values: V(st) — 当前状态价值 - next_values: V(st+1) — 下一状态价值 - dones: 是否episode结束 - gamma: γ — 折扣因子

返回值： - TD目标值 Gt

1def compute_td_targets(
2    rewards: torch.Tensor,
3    values: torch.Tensor,
4    next_values: torch.Tensor,
5    dones: torch.Tensor,
6    gamma: float,
7) -&gt; torch.Tensor:
8    td_targets = rewards + gamma * next_values * (1 - dones)
9    return td_targets

返回值： - 优势函数估计 A(st,at)
