近端策略优化 (PPO) - Proximal Policy Optimization

本文件是蘑菇书EasyRL第五章的PyTorch实现， 参考论文Proximal Policy Optimization Algorithms。

PPO是一种基于策略梯度的强化学习算法。简单的策略梯度方法每个样本只做一次梯度更新， 如果对同一个样本做多次梯度更新会导致策略偏离过大，产生糟糕的策略。 PPO通过限制新旧策略之间的差异，使得我们可以对同一批样本做多次梯度更新。 它通过裁剪梯度流来实现这一目标，如果更新后的策略与采样时的策略差异过大，就会裁剪梯度。

策略梯度基础

在策略梯度方法中，我们希望最大化策略的期望累积奖励：

maxθJ(πθ)=𝔼τ~πθ[∑t=0∞γtrt]

其中： - rt 是时刻 t 的奖励 - πθ 是参数为 θ 的策略 - τ=(s0,a0,s1,a1,…) 是从策略中采样的一条轨迹 - γ∈[0,1] 是折扣因子

轨迹的概率为：

pθ(τ)=p(s1)∏t=1Tπθ(at|st)p(st+1|st,at)

根据log-derivative技巧 ∇f(x)=f(x)∇logf(x)，策略梯度可以写为：

∇θJ(πθ)=𝔼τ~πθ[∑t=0∞γtAπθ(st,at)∇θlogπθ(at|st)]

其中 Aπ(s,a)=Qπ(s,a)−Vπ(s) 是优势函数。

重要性采样与异策略学习

我们可以使用重要性采样将期望从 πθ 转移到另一个策略 πθ′：

𝔼x~p[f(x)]=𝔼x~q[f(x)p(x)q(x)]

应用重要性采样，策略梯度可以改写为：

∇θJ(πθ)=𝔼(st,at)~πθ′[πθ(at|st)πθ′(at|st)Aθ′(st,at)∇θlogπθ(at|st)]

这让我们可以使用旧策略 πθ′ 采集的数据来更新新策略 πθ。

PPO目标函数

PPO1 (带KL惩罚)

PPO1在目标函数中添加KL散度惩罚项：

JPPOθk(θ)=Jθk(θ)−βKL(θ,θk)

其中：

Jθk(θ)≈∑(st,at)πθ(at|st)πθk(at|st)Aθk(st,at)

β 是自适应调整的参数。

PPO2 (裁剪版本)

PPO2使用裁剪的目标函数：

JPPO2θk(θ)≈∑(st,at)min(πθ(at|st)πθk(at|st)Aθk(st,at),clip(πθ(at|st)πθk(at|st),1−ε,1+ε)Aθk(st,at))

其中 ε 是超参数（通常取0.1~0.2）。

裁剪操作确保新旧策略的比率 rt(θ)=πθ(at|st)πθk(at|st) 不会偏离1太多，从而限制了策略更新的幅度。

优势函数估计

优势函数 A(st,at) 可以通过多种方式估计：

  1. 蒙特卡洛估计：A(st,at)=Gt−V(st)，其中 Gt=∑k=t+1Tγk−t−1rk
  2. TD残差：A(st,at)=rt+γV(st+1)−V(st)
  3. GAE (Generalized Advantage Estimation)：A^tGAE=∑l=0∞(γλ)lδt+l

其中 δt=rt+γV(st+1)−V(st) 是TD残差。

PPO裁剪损失

PPO2的核心：裁剪策略比率以限制策略更新幅度。

目标函数：

JPPO2(θ)=𝔼t[min(rt(θ)A^t,clip(rt(θ),1−ε,1+ε)A^t)]

其中： - rt(θ)=πθ(at|st)πθold(at|st) 是策略比率 - A^t 是优势函数的估计 - ε 是裁剪参数

推导过程：

J(πθ)−J(πθold)&=𝔼τ~πθ[∑t=0∞γtAπold(st,at)]&=11−γ𝔼s~dπθ,a~πθ[Aπold(s,a)]&=11−γ𝔼s~dπθold,a~πθold[πθ(a|s)πθold(a|s)Aπold(s,a)]&≈11−γℒCPI

其中 ℒCPI 是CPI (Conservative Policy Iteration) 的目标函数。

计算PPO裁剪损失。

参数映射： - log_pi: logπθ(at|st) — 当前策略的对数概率 - sampled_log_pi: logπθold(at|st) — 旧策略的对数概率 - advantage: A^t — 优势函数估计 - clip: ε — 裁剪参数

裁剪的价值函数损失

类似于策略的裁剪，价值函数的更新也需要裁剪以防止过大变化。

VCLIPπθ(st)=Vπθold(st)+clip(Vπθ(st)−Vπθold(st),−ε,+ε)
ℒVF(θ)=12𝔼t[max((Vπθ(st)−Rt)2,(VCLIPπθ(st)−Rt)2)]

裁剪确保价值函数 Vθ 不会显著偏离 Vθold。

计算裁剪的价值函数损失。

PPO网络

共享特征提取器的Actor-Critic网络结构。 Actor输出策略 πθ(a|s)，Critic输出价值 Vθ(s)。

网络结构：

Q(s,a)=V(s)+(A(s,a)−1|𝒜|∑a′A(s,a′))

但PPO通常使用独立的Actor和Critic头： - Actor: 特征 → 策略分布 πθ(·|s) - Critic: 特征 → 价值估计 Vθ(s)

评估给定状态-动作对的对数概率和价值。

用于PPO更新时计算 logπθ(at|st)。

1class PPONetwork(nn.Module):
2
3    def __init__(self, state_dim: int, action_dim: int, hidden_dim: int = 256):
4        super().__init__()
5 # 共享特征提取器
6        self.feature = nn.Sequential(
7            nn.Linear(state_dim, hidden_dim),
8            nn.ReLU(),

12
13 # Actor头：输出每个动作的对数概率（用于分类动作空间）
14        self.actor = nn.Linear(hidden_dim, action_dim)

17        self.critic = nn.Linear(hidden_dim, 1)
18
19    def forward(self, state: torch.Tensor):
20 # 提取特征
21        features = self.feature(state)
22
23 # Actor: 输出动作 logits
24        action_logits = self.actor(features)
25        action_probs = torch.softmax(action_logits, dim=-1)
26
27 # Critic: 输出状态价值
28        state_value = self.critic(features).squeeze(-1)
29
30        return action_probs, state_value
31
32    def evaluate(self, state: torch.Tensor, action: torch.Tensor):
33        features = self.feature(state)
34        action_logits = self.actor(features)
35        state_value = self.critic(features).squeeze(-1)
36
37 # 计算动作的对数概率
38        action_probs = torch.softmax(action_logits, dim=-1)
39        action_log_probs = torch.log(action_probs + 1e-8)
40
41 # 选择给定动作的对数概率
