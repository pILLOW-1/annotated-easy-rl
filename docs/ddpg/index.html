深度确定性策略梯度 (DDPG) - Deep Deterministic Policy Gradient

本文件是蘑菇书EasyRL第十二章的PyTorch实现，

参考论文Continuous control with deep reinforcement learning。

DDPG是DQN在连续动作空间的扩展，结合了DQN和确定性策略梯度的优点。

确定性策略梯度

在连续动作空间中，动作 $a \in \mathcal{A} \subseteq \mathbb{R}^d$ 是连续值。

确定性策略 $\mu(s)$ 直接输出动作，而不是动作的概率分布。

确定性策略梯度定理：

$$\nabla_\theta J \approx \mathbb{E}_{s \sim \rho^\mu}\left[\nabla_a Q(s, a; \phi)|_{a=\mu_\theta(s)} \nabla_\theta \mu_\theta(s)\right]$$

其中：

  • $\mu_\theta(s)$ 是确定性策略（Actor）
  • $Q(s, a; \phi)$ 是动作价值函数（Critic）
  • $\rho^\mu$ 是策略 $\mu$ 下的状态分布

DDPG算法

DDPG使用四个神经网络：

  1. Actor $\mu_\theta(s)$：确定性策略，直接输出动作
  2. Critic $Q_\phi(s, a)$：动作价值函数
  3. Target Actor $\mu_{\theta'}(s)$：目标策略网络
  4. Target Critic $Q_{\phi'}(s, a)$：目标价值网络

Critic更新

Critic通过最小化TD误差来学习：

$$L(\phi) = \mathbb{E}_{(s,a,r,s') \sim D}\left[(Q_\phi(s, a) - y)^2\right]$$

其中目标值：

$$y = r + \gamma Q_{\phi'}(s', \mu_{\theta'}(s'))$$

Actor更新

Actor通过最大化Critic的输出来学习：

$$\nabla_\theta J \approx \mathbb{E}_{s \sim D}\left[\nabla_a Q_\phi(s, a)|_{a=\mu_\theta(s)} \nabla_\theta \mu_\theta(s)\right]$$

等价于最大化：

$$J \approx \mathbb{E}_{s \sim D}\left[Q_\phi(s, \mu_\theta(s))\right]$$

目标网络软更新

DDPG使用软更新来平滑地更新目标网络：

$$\theta' \leftarrow \tau \theta + (1 - \tau) \theta'$$ $$\phi' \leftarrow \tau \phi + (1 - \tau) \phi'$$

其中 $\tau \ll 1$（通常取0.001~0.01）。

探索策略

DDPG使用Ornstein-Uhlenbeck (OU) 过程来添加探索噪声：

$$dx_t = \theta(\mu - x_t)dt + \sigma dW_t$$

其中：

  • $\theta$ 是均值回归速度
  • $\mu$ 是均值
  • $\sigma$ 是波动率
  • $W_t$ 是维纳过程（布朗运动）

OU过程产生的噪声具有时间相关性，适合连续控制任务。

蘑菇书中的DDPG

根据蘑菇书第十二章，DDPG的核心组件：

  1. Actor网络：输入状态，输出连续动作
  2. Critic网络：输入状态和动作，输出Q值
  3. 经验回放缓冲区：存储 $(s, a, r, s', done)$ 元组
  4. 目标网络：Actor和Critic各有对应的目标网络

训练流程：

  1. Actor根据当前策略 $\mu_\theta(s)$ 选择动作，加上OU噪声
  2. 执行动作，观察奖励和下一状态
  3. 存储经验到回放缓冲区
  4. 从缓冲区采样小批量数据
  5. 更新Critic：最小化TD误差
  6. 更新Actor：最大化Q值
  7. 软更新目标网络

Actor网络（确定性策略）

Actor $\mu_\theta(s)$ 是一个确定性策略网络，

输入状态 $s$，输出连续动作 $a$。

$$a = \mu_\theta(s)$$

输出通常使用tanh激活函数，将动作限制在 $[-1, 1]$ 范围内，

然后缩放到实际的动作空间。

DDPG损失函数

Critic损失

$$L(\phi) = \mathbb{E}_{(s,a,r,s') \sim D}\left[(Q_\phi(s, a) - y)^2\right]$$

其中目标值：

$$y = r + \gamma Q_{\phi'}(s', \mu_{\theta'}(s')) \times (1 - \text{done})$$

Actor损失

Actor通过最大化Critic的输出来学习：

$$J(\theta) = \mathbb{E}_{s \sim D}\left[Q_\phi(s, \mu_\theta(s))\right]$$

由于优化器执行最小化，Actor损失为：

$$L(\theta) = -\mathbb{E}_{s \sim D}\left[Q_\phi(s, \mu_\theta(s))\right]$$

2
3    def __init__(self, gamma: float):
4        super().__init__()
5        self.gamma = gamma
6
7    def compute_critic_loss(
8        self,
9        q_values: torch.Tensor,
10        next_q_values: torch.Tensor,
11        rewards: torch.Tensor,
12        dones: torch.Tensor,
13    ) -&gt; torch.Tensor:
14        &quot;&quot;&quot;
15        计算Critic损失。
16
17        参数映射：
18        - `q_values`: $Q_\phi(s, a)$ — 当前状态-动作对的Q值
19        - `next_q_values`: $Q_{\phi&#x27;}(s', \mu_{\theta'}(s'))$ — 目标网络对下一状态的Q值
20        - `rewards`: $r$ — 奖励
21        - `dones`: 是否episode结束
22
23        返回值：
24        - Critic损失
25        &quot;&quot;&quot;
26        # 目标值：$y = r + \gamma Q_{\phi'}(s', \mu_{\theta'}(s')) \times (1 - \text{done})$
27        with torch.no_grad():
28            target_q = rewards + self.gamma * next_q_values * (1 - dones)
29
30        # MSE损失：$L(\phi) = \mathbb{E}[(Q_\phi(s, a) - y)^2]$
31        critic_loss = nn.MSELoss()(q_values, target_q)
32
33        return critic_loss
34
35    def compute_actor_loss(self, q_values: torch.Tensor) -&gt; torch.Tensor:
36        &quot;&quot;&quot;
37        计算Actor损失。
38
39        参数：
40        - `q_values`: $Q_\phi(s, \mu_\theta(s))$ — Critic对Actor输出动作的Q值
41
42        返回值：
43        - Actor损失（取负号）
44        &quot;&quot;&quot;
45        # 最大化Q值等价于最小化负Q值
46        # $L(\theta) = -\mathbb{E}[Q_\phi(s, \mu_\theta(s))]$
47        actor_loss = -q_values.mean()
48        return actor_loss

Ornstein-Uhlenbeck噪声过程

OU过程用于生成时间相关的探索噪声：

$$dx_t = \theta(\mu - x_t)dt + \sigma dW_t$$

其中：

  • $\theta$：均值回归速度
  • $\mu$：均值
  • $\sigma$：波动率
  • $W_t$：维纳过程

OU过程的特性：

  1. 均值回归：噪声会趋向于均值 $\mu$
  2. 时间相关性：连续时间步的噪声相关
  3. 适合连续控制：噪声变化平滑
1class OUNoise:
2
3    def __init__(
4        self,
5        action_dim: int,
6        mu: float = 0.0,
7        theta: float = 0.15,
8        sigma: float = 0.2,
9    ):
10        self.action_dim = action_dim
11        self.mu = mu
12        self.theta = theta
13        self.sigma = sigma
14        self.state = np.ones(self.action_dim) * self.mu
15
16    def reset(self):
17        &quot;&quot;&quot;重置噪声状态。&quot;&quot;&quot;
18        self.state = np.ones(self.action_dim) * self.mu
19
20    def sample(self) -&gt; np.ndarray:
21        &quot;&quot;&quot;
22        采样OU噪声。
23
24        $$dx = \theta(\mu - x) + \sigma \mathcal{N}(0, 1)$$
25
26        返回值：
27        - OU噪声样本
28        &quot;&quot;&quot;
29        x = self.state
30        dx = self.theta * (self.mu - x) + self.sigma * np.random.randn(len(x))
31        self.state = x + dx
32        return self.state

目标网络软更新

$$\theta' \leftarrow \tau \theta + (1 - \tau) \theta'$$

软更新使目标网络平滑地跟踪主网络，避免剧烈变化。

参数：

  • target: 目标网络 $\theta'$
  • source: 主网络 $\theta$
  • tau: $\tau$ — 软更新系数（通常取0.001~0.01）
1def soft_update(target: nn.Module, source: nn.Module, tau: float):
2    for target_param, param in zip(target.parameters(), source.parameters()):
3        target_param.data.copy_(
4            target_param.data * (1.0 - tau) + param.data * tau
5        )
